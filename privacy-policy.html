Still Alive - 隐私政策

最后更新日期: 2026年2月7日 | 版本: 1.0

简而言之：您的数据仅存储在您的设备上，我们不会与第三方分享您的个人信息。唯一的网络通信是向您指定的紧急联系人发送邮件通知。

1. 我们收集的信息

1.1 本地存储的数据

以下信息存储在您的设备本地，不会上传到我们的服务器：

  • 您的姓名：用于在邮件中称呼您
  • 紧急联系人邮箱：未签到时发送通知的邮箱地址
  • 签到记录：您的历史签到时间戳
  • 设置偏好：签到提醒间隔、通知时间、自动打卡设置等
  • 设备标识符：本地生成的设备 ID（格式：android_xxx 或 uuid_xxx）

1.2 发送到后端的信息

当您的应用向后端服务发送邮件请求时，会包含以下信息：

  • 设备 ID：用于后端限流保护，防止滥用
  • 紧急联系人邮箱：邮件收件人地址
  • 您的姓名：邮件中称呼您
  • 未签到天数：距离上次签到的时间（用于邮件内容）
  • 请求类型：reminder（未签到提醒）或 setup（紧急联系人设置通知）

后端服务说明：

  • 后端服务由 Cloudflare Workers 托管
  • 仅用于发送邮件，不存储任何个人信息
  • 实施了限流保护机制

2. 信息的使用方式

我们收集的信息仅用于以下目的：

  • 发送安全通知：当您超过设定时间未签到时，向您的紧急联系人发送邮件
  • 应用功能：提供签到提醒、历史记录查看等核心功能
  • 防止滥用：设备 ID 用于后端限流，防止恶意使用邮件服务

我们不会将您的信息用于：

  • 广告投放
  • 用户行为分析
  • 销售或共享给第三方
  • 除邮件发送外的任何其他用途

3. 信息共享

我们不会向以下第三方分享您的个人信息：

  • 广告公司
  • 数据分析公司
  • 任何商业或营销目的的第三方

唯一的例外：

  • 当您需要发送邮件通知时，您的姓名会通过加密 HTTPS 连接发送给您指定的紧急联系人（通过后端邮件服务）

4. 数据存储与安全

4.1 本地存储

  • 所有个人数据存储在您的设备本地（使用 Android DataStore）
  • 数据在应用卸载时会被清除
  • 没有云端备份或同步

4.2 安全措施

  • 后端通信使用 HTTPS 加密
  • 设备 ID 在本地生成，不收集设备硬件信息
  • 应用代码开源，可审计
  • 不包含第三方分析 SDK 或追踪工具

5. 应用权限说明

本应用请求以下 Android 权限，并仅在必要时使用：

权限 用途
INTERNET 向后端服务发送邮件请求（仅用于发送邮件）
POST_NOTIFICATIONS 显示每日签到提醒通知
SCHEDULE_EXACT_ALARM 设置准确的定时提醒
RECEIVE_BOOT_COMPLETED 设备重启后恢复定时提醒任务
ACTIVITY_RECOGNITION 检测步数以实现自动打卡
FOREGROUND_SERVICE + FOREGROUND_SERVICE_HEALTH 步数检测服务运行

注意：您可以随时在系统设置中撤销这些权限，但可能会影响应用功能。

6. 儿童隐私

本应用不针对 13 岁以下儿童。我们不会故意收集儿童的个人信息。

7. 数据删除

您可以随时通过以下方式删除您的数据：

  • 卸载应用：清除所有本地存储的数据
  • 清除应用数据：在系统设置中选择 "清除数据"

8. 隐私政策的变更

我们可能会不时更新本隐私政策。重大变更时，我们会通过应用内通知或更新说明告知您。

9. 联系我们

如果您对本隐私政策有任何疑问或建议，请通过以下方式联系：

  • GitHub Issues: https://github.com/twiceyuan/StillAlive/issues

附录：数据处理的法律依据

根据适用的数据保护法律（如 GDPR、CCPA 等），我们处理您的个人信息的法律依据为：

  • 履行合同：提供您请求的服务（签到、邮件通知）
  • 合法利益：防止滥用后端服务（限流）
  • 用户同意：处理您自愿提供的个人信息
